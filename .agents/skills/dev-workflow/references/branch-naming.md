# 브랜치 & 커밋 규칙

## 브랜치
```
{type}/issue-{N}-{설명}
```
- 설명: kebab-case, 영어, 3단어 이내, 소문자

## 커밋
```
{type}: {description} (#{N})
```

## Type

| Type | 용도 |
|------|------|
| `feat` | 새 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 변경 |
| `style` | 코드 포맷팅 (기능 변경 없음) |
| `refactor` | 리팩토링 (기능 변경 없음) |
| `test` | 테스트 추가/수정 |
| `chore` | 빌드, 설정 변경 |
